# 介绍

**LangChain** 是一个基于语言模型开发应用程序的框架。它使得应用程序能够：

- **具有上下文感知能力** 将语言模型连接到上下文来源（提示说明、少量示例、内容以确定其响应等）。
- **推理能力**：依赖语言模型进行推理（根据提供的上下文如何回答、采取什么行动等）。

这个框架包括几个部分。

- **LangChain 库**： Python 和 JavaScript 库。包含各种组件的接口和集成，用于将这些组件组合成链和代理的基本运行时，以及链和代理的现成实现。
- **[LangChain 模板](https://python.langchain.com/docs/templates)**：一组易于部署的参考架构，适用于各种任务。 (仅限 Python)
- **[LangServe](https://python.langchain.com/docs/langserve)**：用于将 LangChain 链部署为 REST API 的库。 (仅限 Python)
- **[LangSmith](https://smith.langchain.com/)**：一个开发者平台，让您可以在任何 LLM 框架上调试、测试、评估和监控构建的链，并与 LangChain 实现无缝集成。


![LangChain Diagram](/img/langchain_stack_feb_2024.webp)

这些产品共同简化了整个应用程序的生命周期：
- **开发**：使用 LangChain/LangChain.js 编写您的应用程序。使用模板作为参考，立即上手。
- **生产**：使用 LangSmith 来检查、测试和监控您的链，以便您不断改进并放心地部署
- **部署**：使用 LangServe 将任何链转换为 API。

## LangChain 库

LangChain 包的主要价值主张包括：

1. **组件**：可组合的工具和集成，用于处理语言模型。组件是模块化的，无论您是否使用 LangChain 框架的其他部分，都很容易使用
2. **现成的链**：内置的组件集合，用于完成更高级别的任务。

现成的链使得快速入门变得容易。组件使得定制现有链和构建新链变得容易。

## LangChain 表达式语言 (LCEL)

LCEL 是一种声明式的链式组合方式。从一开始，LCEL 就被设计成支持将原型投入生产使用，无需进行代码更改，从最简单的“提示 + LLM”链到最复杂的链。

- **[概览](/docs/expression_language/)**：LCEL 及其优势
- **[接口](/docs/expression_language/interface)**：LCEL 对象的标准接口
- **[操作指南](/docs/expression_language/how_to/routing)**：LCEL 的关键功能
- **[示例集](/docs/expression_language/cookbook)**：完成常见任务的示例代码
